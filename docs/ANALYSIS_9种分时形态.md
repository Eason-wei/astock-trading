# 9 种分时形态系统深度分析

> **源码范围**：`types.py` + `classifier.py` + `predictor.py` + `morphology_config.json`
> **分析日期**：2026-04-23
> **分析方法**：逐行交叉验证分类逻辑、预测规则、阶段覆盖

---

## 一、架构总览

```
241点分钟数据 / OHLC数据
       ↓
  extract_features()          ← classifier.py:32-111
  提取 MorphologyFeatures（13个字段）
       ↓
  classify()                  ← classifier.py:222-276
  基于规则树 → Morphology 枚举（9种之一）
       ↓
  predict()                   ← predictor.py:114-153
  形态 × 市场阶段 → T+1 预测结论
       ↓
  { direction, confidence, expected_change, warnings }
```

**核心设计原则**：
- **分类器只负责"这是什么形态"**，不含任何预测逻辑
- **预测引擎负责"T+1会怎样"**，形态×阶段的组合决定结论
- **置信度从 AccuracyTracker 实时查询**（贝叶斯平滑），而非 JSON 静态值

## 二、9种形态逐一分析

### 2.1 A类一字板 — 最强形态

| 属性 | 值 |
|------|-----|
| **分类条件** | `board_quality == '一字板'` |
| **判定细节** | `close_pct < 1 AND amplitude < 3`（几乎没波动） |
| **T+1方向** | positive（冰点/升温/修复期）|
| **基础置信度** | 0.85 |
| **风险等级** | 极低 |

**分类逻辑**：`classifier.py:238-239`
```python
if f.board_quality == '一字板':
    return Morphology.A
```
**优先级最高**——在所有其他判断之前检查。

**预测逻辑**（`predictor.py:221-251`）：
- **降温/退潮/高潮期** → 降权为 neutral（conf=0.45），警告"高开低走风险大"
- **冰点/升温/修复期** → 维持 positive（conf=0.85）
- 额外检查：`q4_volume_pct > 20` 时追加警告"尾盘放量，T+1可能开板"

**⚠️ 设计亮点**：A类不是无条件看多。退潮期的一字板往往是"最后一棒"，次日高开低走概率极大。这个阶段区分非常实战。

### 2.2 B类正常涨停 — 稳健型

| 属性 | 值 |
|------|-----|
| **分类条件** | `close_pct >= 9.5 AND amplitude < 8` |
| **T+1方向** | positive（通用）/ neutral（高潮期覆盖）|
| **基础置信度** | 0.65 |
| **风险等级** | 中低 |

**分类逻辑**：`classifier.py:242-243`
```python
if f.close_pct >= 9.5 and f.amplitude < 8:
    return Morphology.B
```

**关键约束**：`amplitude < 8` 排除了"冲高回落再回封"的烂板。B类要求盘中走势稳健。

**预测逻辑**：
- 通用路径：positive, conf=0.65, 预期 "+1%~+3%"
- **高潮期 JSON 覆盖**：→ neutral, conf=0.55, 预期 "-2%~+3%"
  - 规则描述："高潮期次日B类降级为中性，高开亦谨慎"
- 板块共振加成：`sector_strength > 0.7` 时 +15% confidence

**⚠️ 实战意义**：B类的 amplitude<8 这个阈值很关键。如果一只股票涨停但振幅超过8%，它会走入 C1 或 E2 的分支，不会被归为 B 类。这个过滤有效地把"烂板"排除在了 B 类之外。

### 2.3 C1冲高回落 — 高风险形态

| 属性 | 值 |
|------|-----|
| **分类条件** | `high_pct - close_pct > 5 AND amplitude > 10` |
| **T+1方向** | negative（通用）/ **positive（升温/高潮期覆盖！）** |
| **基础置信度** | 0.75 |
| **风险等级** | 高 |

**分类逻辑**：`classifier.py:246-247`
```python
if f.high_pct - f.close_pct > 5 and f.amplitude > 10:
    return Morphology.C1
```

**物理含义**：最高点比收盘点多 5 个点以上，且全天振幅超 10% = 典型的"冲高大幅回落"。比如冲到 +8%，收盘只有 +2%。

**🔴 这是最有争议的形态——原规则被数据推翻了！**

| 版本 | 规则 | 来源 |
|------|------|------|
| **原始硬编码** | "C1 × 高潮期 → 必跌（高潮地雷）" | predictor.py 注释 L172-176 |
| **JSON 覆盖（升温期）** | → **positive**, conf=0.62, "+8%~+15%" | 43样本/胜率95.3%/avg=+10.68% |
| **JSON 覆盖（高潮期）** | → **positive**, conf=0.62, "+8%~+18%" | 105样本/胜率100%/avg=+11.92% |

predictor.py 源码中保留了历史注释：
```python
# 规则1：C1 × 高潮期 → 已迁移到JSON override
# 数据验证显示C1×高潮期胜率100%，原规则错误，已禁用
```

**💡 深层原因**：C1 在升温/高潮期往往是因为"板块内部轮动"——早盘资金先拉了一波，然后获利盘涌出导致回落，但题材本身还在发酵。T+1 如果情绪延续，反而可能修复。这与直觉完全相反，是**数据驱动修正规则的典型案例**。

### 2.4 D1低开低走 — 弱势形态

| 属性 | 值 |
|------|-----|
| **分类条件** | `open_pct < -2 AND close_pct < open_pct` |
| **T+1方向** | negative（通用）/ neutral（高潮期）/ negative加重（退潮期）|
| **基础置信度** | 0.70 |
| **风险等级** | 高 |

**分类逻辑**：`classifier.py:250-251`
```python
if f.open_pct < -2 and f.close_pct < f.open_pct:
    return Morphology.D1
```

**物理含义**：开盘就跌 2% 以上，而且越走越弱（收盘价比开盘价还低）。典型的弱势股。

**预测逻辑的阶段分化**：

| 阶段 | 方向 | 置信度 | 预期 | 说明 |
|------|------|--------|------|------|
| 通用 | negative | 0.70 | "-1%~+1%" | 弱势延续 |
| **高潮期覆盖** | **neutral** | 0.45 | "-2%~+4%" | "逆势错杀股，或有修复机会" |
| **退潮期覆盖** | **negative 加重** | 0.78 | "<-2%" | "退潮期延续弱势，加权看空" |

**⚠️ 高潮期的 D1 反而是机会？** 这听起来反直觉，但逻辑是：高潮期市场整体强势，个别低开的可能是被错杀的，次日有修复预期。而退潮期的 D1 则是真弱，雪上加霜。

### 2.5 D2尾盘急拉 — 可疑形态 ⚠️

| 属性 | 值 |
|------|-----|
| **分类条件** | `f30 > 80 AND -2 <= close_pct <= 1` |
| **T+1方向** | negative（高潮/退潮/降温）/ neutral（其他阶段）|
| **基础置信度** | 0.80 |
| **风险等级** | 高 |

**分类逻辑**：`classifier.py:254-255`
```python
if f.f30 > 80 and -2 <= f.close_pct <= 1:
    return Morphology.D2
```

**物理含义**：前 30 分钟成交量占全天 80% 以上（说明尾盘几乎没有量），且收盘只在 -2% 到 +1% 之间（不涨不跌或微涨微跌）。

等等——这里有个需要仔细理解的地方：**f30 > 80 表示前30分钟成交占比极高，那尾盘怎么急拉？**

实际上 D2 的典型分时是这样的：
- 开盘后快速放量拉升（所以 f30 很高）
- 之后一路缩量横盘或阴跌
- 尾盘没有真正的急拉动作
- **"尾盘急拉"这个名字有点误导**——它更准确的描述应该是"早盘脉冲后无力"

但再看 predictor.py 中 D2 的特殊规则（L195-218），它确实在检查 `features.f30 > 80`，并且对高潮/退潮/降温期给出 negative 判断，理由是"尾盘偷袭在退潮期次日大概率低开杀"。

**⚠️ 分类条件和预测条件的重复检查**：
- classify() 已经确保了 D2 的 `f30 > 80`
- predict() 的 `_apply_special_rules()` 又检查了一遍 `features.f30 > 80`
- 这是**冗余但不有害**的防御性编程

**D2 的阶段分化很清晰**：

| 阶段 | 方向 | 置信度 | 预期 | 警告 |
|------|------|--------|------|------|
| 高潮/退潮/降温 | **negative** | 0.80 | "<-3%" | "尾盘偷袭次日大概率低开杀" |
| 升温/修复/冰点 | neutral | 0.55 | "-1%~+2%" | "降级为中性" |

### 2.6 E1普通波动 — 方向不明（默认兜底）

| 属性 | 值 |
|------|-----|
| **分类条件** | `amplitude < 5` 且不符合上述任何类型 |
| **T+1方向** | neutral（通用）/ **positive（多阶段覆盖！）** |
| **基础置信度** | 0.45 |
| **风险等级** | 中 |

**分类逻辑**：`classifier.py:272-276`
```python
# H：横向整理（更严格的条件先检查）
if f.amplitude < 2 and f.q1_volume_pct > 70:
    return Morphology.H

# E1：普通波动（最终兜底）
if f.amplitude < 5:
    return Morphology.E1

# 默认也是 E1
return Morphology.E1
```

E1 是**兜底形态**——振幅小于 5%、又不是涨停、又不是横向整理的股票，全部归入 E1。它是数量最大的类别。

**🔥 E1 是阶段覆盖最多的形态——5个阶段中有4个做了覆盖！**

| 阶段 | 方向 | 置信度 | 预期 | 数据依据 |
|------|------|--------|------|---------|
| 通用 | neutral | 0.45 | "-2%~+2%" | 无 |
| **升温期** | **positive** | 0.60 | "+3%~+12%" | 60样本/avg=+8.97%/胜率93.3% |
| **高潮期** | **positive** | 0.55~0.62 | "+3%~+12%" | 29样本/avg=+4.50%/胜率69% |
| **冰点期** | **positive** | 0.60 | "+2%~+8%" | 冰点日抛压轻+情绪反转 |
| **修复期** | **positive** | 0.54 | "+2%~+8%" | 继承冰点期配置 |

**💡 核心洞察**：E1 在通用情况下确实是"方向不明"（neutral, 0.45），但在**特定市场阶段**它变成了正向信号。原因是：
- **升温/高潮期**：市场整体强势，普通波动股跟随大盘水涨船高
- **冰点期**：抛压已经出清，次日报复性反弹
- **修复期**：同冰点期逻辑

这说明 **E1 不是真的"无信号"，而是"条件性信号"**。它的方向取决于市场环境。

### 2.6.1 🔴 E1×高潮期 — 实测失败（2026-04-23 验证记录）

**实测结果 vs 预期：**

| 项目 | 预期（JSON） | 实际 |
|------|-------------|------|
| 方向 | positive | ❌ negative |
| 置信度 | 0.55~0.62 | — |
| 预期涨幅 | +3%~+12% | **-1.9%** |
| 胜率（历史） | 69%（29样本） | 失败 |

**根本矛盾**：JSON 配置说高潮期 E1 应该 positive，但 step7/8 验证捕获到 -1.9% 的 undershoot。mechanism 字段当时只记录了"形态有效性需重新评估"，这是表层归因。

**深层因果追溯（causal_chains.json 待补全）：**

当前机制描述 `E1×高潮期→positive` 过于粗糙。真实市场中有**两种完全不同的 E1**：

1. **E1-A：随波逐流型**（amplitude 接近 5% 边界）
   - 大盘涨它跟着涨，大盘跌它跟着跌
   - 高潮期若大盘冲高回落，这类股票 T+1 直接跟跌
   - 机制：高潮期情绪高潮次日本身就面临兑现压力，E1-A 没有基本面/题材支撑，只有大盘动量

2. **E1-B：蓄势整理型**（amplitude 偏小，接近 H 的边界）
   - 主力控盘迹象，振幅极窄
   - 可能在酝酿方向，但当前没信号
   - 这类在高情绪周期反而可能补涨

**E1×高潮期胜率 69% 的数据背后**：这 29 个样本很可能混合了 E1-A 和 E1-B，两类方向相反。E1-B 的正收益拉高了均值，但如果 E1-A 占比更高，或买入时机恰好在大盘高点，结果就是负收益。

**真实机制**：

```
E1×高潮期 失效条件：
  E1-A（amplitude 偏高，3~5%）+ 大盘高位 → 负收益
  原因：高潮期次日情绪回落，E1-A 没有支撑直接杀

E1×高潮期 有效条件：
  E1-B（amplitude 偏低，<2%）+ 非大盘高点 → 正收益
  原因：控盘整理股趁高潮期氛围补涨
```

**修正建议**：

| 方案 | 改动量 | 效果 |
|------|--------|------|
| **A. amplitude 阈值切分** | 改 classifier.py 加一个 E1-1/E1-2 子类，amplitude ≥3 为 E1-A，<3 为 E1-B | 精细化，但需重新采集样本 |
| **B. 动态 amplitude 权重** | 在 predictor 中对 E1×高潮期引入 amplitude 修正，amplitude>3 时降权至 neutral | 不改分类器，只改预测逻辑 |
| **C. 降级 E1×高潮期置信度** | 直接把 JSON 中的 confidence 从 0.55~0.62 降到 0.45，方向改为 neutral | 最保守，但丢失正收益机会 |

**推荐方案 B**——在 `_build_from_override()` 或 `_build_generic_prediction()` 中，对 E1×高潮期加一条 amplitude 修正：

```python
# 高潮期 E1：amplitude ≥ 3 时降权
if morphology == Morphology.E1 and market_stage == '高潮期':
    if features.amplitude >= 3:
        direction = 'neutral'
        confidence = min(confidence, 0.45)
        warnings.append("E1×高潮期amplitude≥3，降级为中性")
```

**待验证**：需要从历史数据中分离 amplitude ≥3 和 <3 的 E1 样本，分别统计 T+1 胜率和均值，作为修正依据。当前 29 样本未做 amplitude 维度的拆分，这是后续验证的关键任务。

### 2.8 F1温和放量稳步推进 — 最安全形态 🏆

| 属性 | 值 |
|------|-----|
| **分类条件** | `40<=q1_volume_pct<=60 AND 3<=amplitude<=8 AND close_pct>open_pct>0` |
| **T+1方向** | **positive（无条件）** |
| **基础置信度** | **0.88（全场最高）** |
| **风险等级** | 低 |

**分类逻辑**：`classifier.py:258-260`
```python
if 40 <= f.q1_volume_pct <= 60 and 3 <= f.amplitude <= 8:
    if f.close_pct > f.open_pct > 0:
        return Morphology.F1
```

三个条件必须同时满足：
1. **Q1 成交量占比 40-60%**：开盘放量适中，不是巨量（>60%暗示早盘脉冲），也不是缩量（<40 暗示启动晚）
2. **振幅 3-8%**：有明显波动但不过激
3. **收涨且开盘为正**：全天趋势向上，量价配合

**预测逻辑**：F1 是唯一**无条件 positive** 且不走通用路径的形态。
```python
# predictor.py:179-192 —— 无阶段检查，直接返回 positive
if morphology == Morphology.F1:
    return {
        't1_direction': 'positive',
        'confidence': real_conf,
        'rule_applied': 'F1规则：3/3胜率，温和放量稳步推进',
    }
```

**💡 为什么 F1 最安全？**
- 40-60% 的 Q1 占比 = 主力在开盘阶段稳步建仓，不是偷袭（D2）也不是脉冲（q1>60%）
- 3-8% 振幅 = 有足够换手让获利盘消化，但又不会引发恐慌
- 收盘 > 开盘 > 0 = 全天重心上移，多头主导
- 这种分时图通常对应"主力吸筹完毕，准备拉升"的阶段

| **T+1方向** | neutral |
| **基础置信度** | 0.40（全场最低）|
| **风险等级** | 高 |

**分类逻辑**：`classifier.py:268-269`
```python
if f.amplitude < 2 and f.q1_volume_pct > 70:

```

**没有任何阶段覆盖**——H 在所有阶段都是 neutral, 0.40。这符合其"无效信号"的定位。

## 三、分类优先级与边界问题

### 3.1 分类决策树（按代码执行顺序）

```
输入：MorphologyFeatures f
  │
  ├─① board_quality == '一字板' ? ──────────→ A ✅
  │
  ├─② close_pct >= 9.5 AND amplitude < 8 ? ──→ B ✅
  │     （排除：amp≥8 的涨停 → 可能是 C1/E2）
  │
  ├─③ high_pct - close_pct > 5 AND amp > 10 ? → C1 ✅
  │     （冲高大幅回落）
  │
  ├─④ open_pct < -2 AND close_pct < open_pct ? ─→ D1 ✅
  │
  ├─⑤ f30 > 80 AND -2≤close_pct≤1 ? ────────→ D2 ✅
  │
  ├─⑥ 40≤q1_vol≤60 AND 3≤amp≤8 AND 收涨? ───→ F1 ✅
  │     （最窄的条件窗口）
  │
  ├─⑦ amp > 8 AND close_pct < 9.5 ? ─────────→ E2 ✅
  │     （宽幅震荡，非涨停）
  │
  ├─⑧ amp < 2 AND q1_vol > 70 ? ─────────────→ H ✅
  │     （横向整理）
  │
  ├─⑨ amp < 5 ? ─────────────────────────────→ E1 ✅
  │
  └─⑩ 兜底 ──────────────────────────────────→ E1 ✅
```

### 3.2 ⚠️ 边界重叠与潜在误判

| # | 重叠场景 | 先命中 | 被屏蔽 | 影响 |
|---|---------|--------|--------|------|
| **B1** | 一字板 + 振幅刚好 <8 + 涨停 | ① A | B | ✅ 正确，A优先级高于B |
| **B2** | 涨停 + 振幅 7.9%（接近8） | ② B | — | 边界值，amp=8.0 会漏到后面 |
| **B3** | 冲高回落 + 也满足 D2 条件 | ③ C1 vs ⑤ D2 | C1先于D2 | ⚠️ 见下文 |
| **B4** | F1 条件满足 + 也是涨停(amp<8) | ② B vs ⑥ F1 | **B先于F1** | 🔴 见下文 |
| **B5** | E2(amp>8,非涨停) + C1(high-close>5,amp>10) | ③ C1 vs ⑦ E2 | **C1先于E2** | ✅ 正确 |
| **B6** | H(amp<2,q1>70) + E1(amp<5) | ⑧ H vs ⑨ E1 | H先于E1 | ✅ 正确 |

### 3.3 🔴 关键发现：F1 形态永远不会命中涨停股

这是一个**隐蔽的设计缺陷**：

```python
# L242: B类检查在前
if f.close_pct >= 9.5 and f.amplitude < 8:   # ← 涨停 + 振幅<8
    return Morphology.B                          # ← 直接返回 B

# L258: F1 检查在后
if 40 <= f.q1_volume_pct <= 60 and 3 <= f.amplitude <= 8:
    if f.close_pct > f.open_pct > 0:
        return Morphology.F1                      # ← 永远走不到这里（如果是涨停）
```

**问题**：如果一只股票涨停了（close_pct>=9.5），且 Q1 占比 40-60%，振幅 3-8%，它会被归类为 **B 类**而不是 **F1**。

**这意味着什么？**
- F1 的"无条件 positive, conf=0.88"永远不适用于涨停股
- 涨停股只能拿到 B 类的"positive, conf=0.65"（通用路径）或更低（高潮期降级为 neutral）
- **一只温和放量稳步推进并封板的股票，其实比 F1 的定义更安全，但它拿不到 F1 的待遇**

**是否需要修复？** 取决于设计意图：
- 如果 F1 的本意就是"非涨停的稳步推进" → 当前行为正确，无需修改
- 如果 F1 应该包含"温和放量封板" → 需要 B 和 F1 之间加一个分支判断

---

### 3.4 ⚠️ C1 与 D2 的优先级

C1（③）排在 D2（⑤）前面。如果一个股票同时满足两者：
- C1: `high_pct - close_pct > 5 AND amplitude > 10`
- D2: `f30 > 80 AND -2 ≤ close_pct ≤ 1`

同时满足的场景：**冲高大幅回落 + 早盘放量 + 收盘微涨微跌**
→ 典型的"早盘脉冲后大幅回落" → 归入 C1（冲高回落）

这是合理的——C1 的"高风险"属性比 D2 更匹配这种分时特征。

### 4.2 _judge_push_style() 的尴尬地位

`_judge_push_style()`（L113-132）计算了四种拉升方式（早盘脉冲/午盘拉升/尾盘偷袭/全天稳健），结果写入 `push_up_style` 字段。但是：
- **classify() 不读取 push_up_style**
- **predict() 不读取 push_up_style**
- 它是一个"计算了但没人用"的特征

**可能的改进方向**：
- 用 `push_up_style` 增强 D2 的判断（当前只用 f30>80）
- 用 `push_up_style` 区分 B 类的子类型（早盘封板 vs 尾盘回封）

## 五、预测引擎深度分析

### 5.1 预测路径选择流程

```
predict(features, morphology, market_stage)
  │
  ├─ Step 1: 检查 JSON stage_overrides
  │   └─ 有覆盖? → _build_from_override() → 返回 ✅
  │
  ├─ Step 2: _apply_special_rules()
  │   ├─ F1 ? → 无条件 positive, conf=0.88 → 返回 ✅
  │   ├─ D2 ? → 检查阶段 → negative/neutral → 返回 ✅
  │   ├─ A ? → 检查阶段 → positive/neutral → 返回 ✅
  │   └─ None ? → 继续 ↓
  │
  └─ Step 3: _build_generic_prediction()
      └─ 读 JSON morphologies 配置 + 板块加成 → 返回 ✅
```

### 5.2 置信度的双层机制

```
用户请求预测
  ↓
JSON 提供 t1_confidence（先验值，如 0.85）
  ↓
AccuracyTracker._get_real_precision() 查询历史
  ↓
├─ 有 ≥3 个历史样本 → 返回贝叶斯平滑后的真实准确率（替换 JSON 值）
└─ 无足够样本 → fallback 到 JSON 值
```

**这是很好的设计**——JSON 配置提供"专家经验先验"，Tracker 提供"数据验证后验"。随着运行时间增长，系统会自动从"经验驱动"过渡到"数据驱动"。

## 六、发现的问题清单

### 🔴 P0：F1 无法命中涨停股（§3.3 已详述）

## 七、总体评价

### ✅ 做得好的地方

1. **分类与预测分离**：classifier 不管预测，predictor 不管分类——单一职责原则执行得好
2. **置信度双层机制**：JSON 先验 + Tracker 后验的贝叶斯思想很优雅
3. **阶段覆盖体系**：同一形态在不同市场阶段有不同表现，这符合A股"重势不重股"的特点
4. **数据驱动的规则修正**：C1×高潮期从"必跌"改为"必涨"、E2×冰点期从"中性"改为"做多"——都是用真实数据推翻直觉的案例
5. **F1 无条件 positive**：3/3 胜率的形态获得最高置信度，简单明确

### ⚠️ 需要关注的地方

1. **P0 F1-B 边界问题**：最安全的形态无法应用于涨停股
2. **P1 E1×高潮期 amplitude 修正已完成**：predictor 规则1.5 拦截 amplitude≥3，剩余 amplitude 维度样本验证任务
3. **P0 JSON 高潮期重复 key 已修复**：B类/D1/C1/E2/E1 全合并到一个高潮期 block
